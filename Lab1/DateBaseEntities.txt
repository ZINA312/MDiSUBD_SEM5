Сущности:
    1. Пользователь
        id (INT, PK)
        имя (VARCHAR, NOT NULL)
        email (VARCHAR, UNIQUE, NOT NULL)
        пароль (VARCHAR, NOT NULL)
        роль_id (INT, FK)
    2. Роль
        id (INT, PK)
        название (VARCHAR, UNIQUE, NOT NULL)
    3. Товар(Можно реализовать разные таблицы под разные виды товаров)
        id (INT, PK)
        название (VARCHAR, NOT NULL)
        описание (TEXT)
        цена (DECIMAL, NOT NULL)
        категория_id (INT, FK)
    4. Категория
        id (INT, PK)
        название (VARCHAR, UNIQUE, NOT NULL)
    5. Заказ
        id (INT, PK)
        пользователь_id (INT, FK)
        дата (DATETIME, NOT NULL)
        статус (VARCHAR, NOT NULL)
    6. Заказ_Товар
        заказ_id (INT, FK)
        товар_id (INT, FK)
        количество (INT, NOT NULL)
        PRIMARY KEY (заказ_id, товар_id)
    7. Отзыв
        id (INT, PK)
        пользователь_id (INT, FK)
        товар_id (INT, FK)
        текст (TEXT, NOT NULL)
        рейтинг (INT, NOT NULL)
    8. Журнал
        id (INT, PK)
        пользователь_id (INT, FK)
        действие (VARCHAR, NOT NULL)
        дата (DATETIME, NOT NULL)
    9. Способ_доставки
        id (INT, PK)
        название (VARCHAR, NOT NULL)
    10. Способ_оплаты
        id (INT, PK)
        название (VARCHAR, NOT NULL)
    11. Корзина
        id (INT, PK)
        пользователь_id (INT, FK)
        дата_создания (DATETIME, NOT NULL)
        итоговая_сумма (DECIMAL, NOT NULL)
    12. Корзина_Товар
        корзина_id(INT, FK)
        товар_id(INT, FK)
        количество(INT, NOT NULL)

Связи сущностей
    Пользователь и Роль: Один пользователь может иметь одну роль, но одна роль может быть назначена многим пользователям (N:1).
    Товар и Категория: Один товар принадлежит одной категории, но одна категория может быть назначена многим товарам (N:1).
    Заказ и Пользователь: Один заказ принадлежит одному пользователю, но один пользователь может иметь несколько заказов (N:1).
    Заказ и Товар: Один заказ может содержать несколько товаров, а один товар может быть в нескольких заказах (M:N).
    Отзыв и Пользователь: Один пользователь может оставить несколько отзывов (1:N).
    Отзыв и Товар: Один товар может иметь несколько отзывов (1:N).
    Журнал и Пользователь: Один пользователь может иметь несколько записей в журнале (1:N).
    Пользователь и Корзина: Один пользователь может иметь одну корзину(1:1).
    Корзина и Товар: Корзина может содержать множество товаров (1:N).
